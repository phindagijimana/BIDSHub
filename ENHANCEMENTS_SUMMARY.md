## Overview

Enhanced Data Explorer v1.5 with increased dataset capacity, BIDS validation, and custom cohort export functionality.

**Date**: February 21, 2026

## Enhancements Implemented

### 1. ✅ Increased Dataset Limit (3 → 5 datasets)

**Previous Limit**: 3 datasets

**Impact**:

- Better support for complex research scenarios

### 2. ✅ BIDS Validation for All Datasets

**Purpose**: Ensure all datasets follow BIDS specification before being added

**New Module**: `src/bids_validator.py`

**Features**:
- **Required Files Check**: Validates presence of `dataset_description.json`
- **Dataset Description Validation**: Checks for required fields (Name, BIDSVersion)
- **Subject Structure Validation**: Validates `sub-*` directories
- **Session Structure Validation**: Validates `ses-*` directories if present
- **Modality Validation**: Checks for valid modality directories (anat, func, dwi, etc.)
- **Participants.tsv Validation**: Validates format and checks for duplicates
- **Common Issues Detection**: Checks for hidden files, README, CHANGES

**Validation Levels**:
- **Errors**: Critical issues that prevent dataset from being valid BIDS
- **Warnings**: Non-critical issues but recommended fixes

**Integration Points**:
1. **Dataset Addition**: Optional BIDS validation during dataset setup
2. **Checkbox Option**: "Validate BIDS compliance" (default: checked)
3. **Validation Feedback**: Shows detailed errors/warnings
4. **Override Option**: Users can add non-compliant datasets with confirmation

**Example Validation Output**:
```
✅ Dataset is valid BIDS

OR

❌ 2 Error(s):
  - Missing required file: dataset_description.json
  - Missing required field in dataset_description.json: Name

⚠️  3 Warning(s):
  - participants.tsv not found (recommended)
  - README file not found (recommended)
  - No NIfTI files found in sub-001/anat
```

### 3. ✅ Export Custom Cohort as New BIDS Dataset

**Purpose**: Create new BIDS-compliant datasets from filtered subjects across multiple source datasets

**New Module**: `src/cohort_exporter.py`

**Core Functionality**:

#### Step 1: Select Source Datasets
- Multi-select from active datasets
- Shows platform icons (🔐 Pennsieve / 🌍 OpenNeuro)
- Can select 1-5 datasets

#### Step 2: Filter Subjects
- **Optional Metadata Filtering**:
  - Age range filter
  - Sex filter
  - Additional metadata fields
- Shows subject count before and after filtering
- Cross-dataset filtering support

#### Step 3: Select Subjects
- "Select all" option
- Manual multi-select
- Shows subject ID and source dataset
- Real-time count of selected subjects

#### Step 4: Configure Export
- **Cohort Name**: Name for new BIDS dataset
- **Output Directory**: Where to create the dataset
- **Copy Mode**:
  - `symlink`: Fastest, requires source datasets
  - `hardlink`: Fast, space-efficient, independent inodes
  - `copy`: Slowest, fully independent copy
- **Include Derivatives**: Option to include processed data
- **Dataset Description**: Custom description for the cohort

#### Export Results

Creates a complete BIDS-compliant dataset with:

**Required BIDS Files**:
- `dataset_description.json` - Includes source dataset info, export date, generated by info
- `participants.tsv` - Aggregated metadata from source datasets
- `README` - Cohort overview and usage instructions
- Subject directories with data (based on copy mode)

**dataset_description.json Example**:
```json
{
  "Name": "Custom_Cohort_20260221",
  "BIDSVersion": "1.6.0",
  "DatasetType": "derivative",
  "GeneratedBy": [{
    "Name": "Data Explorer Cohort Exporter",
    "Version": "1.5.0",
    "Description": "Custom cohort exported from multiple datasets"
  }],
  "SourceDatasets": [
    {"name": "TrackTBI", "platform": "pennsieve"},
    {"name": "ds000246", "platform": "openneuro"}
  ],
  "ExportDate": "2026-02-21T10:30:00",
  "ExportedSubjects": 25
}
```

**participants.tsv Features**:
- Combines metadata from all source datasets
- Adds `source_dataset` and `source_platform` columns
- Preserves original metadata columns (age, sex, diagnosis, etc.)
- Handles missing metadata gracefully

**Post-Export Validation**:
- Automatically validates exported dataset for BIDS compliance
- Shows warnings if validation issues found
- Provides metrics: subjects exported, total size, output path

## Use Cases

### Use Case 1: Multi-Study Cohort Creation

**Scenario**: Researcher wants to create a cohort of 30 male subjects aged 25-40 from 3 different TBI studies

**Workflow**:
1. Add 3 datasets to Data Explorer (all BIDS validated)
2. Navigate to Export → Export Custom Cohort
3. Select all 3 datasets
4. Apply filters: sex='M', age=25-40
5. Select all 30 filtered subjects
6. Configure: name="TBI_Males_25-40", copy_mode="symlink"
7. Export → New BIDS dataset created in minutes

**Result**: BIDS-compliant dataset with 30 subjects, ready for analysis, with full provenance tracking

### Use Case 2: QC-Passed Subjects Export

**Scenario**: After QC review, export only subjects that passed QC from multiple datasets

**Workflow**:
1. Perform QC on subjects across datasets
2. Navigate to Export → Export Custom Cohort
3. Select datasets
4. In Subjects Browser, filter by QC status='pass'
5. Export filtered list to new cohort
6. Use copy mode for full independence

### Use Case 3: Dataset Validation Before Analysis

**Scenario**: Validate downloaded datasets before running analysis pipeline

**Workflow**:
1. Download datasets via Data Explorer
2. Add new dataset to Data Explorer
3. Enable "Validate BIDS compliance"
4. Review validation report
5. Fix any errors/warnings
6. Re-validate until clean

## Limitations & Known Issues

### Current Limitations

1. **Dataset Limit**: Still capped at 5 (not unlimited)
2. **Copy Modes**: 
   - Symlinks require source datasets to remain accessible
   - Hardlinks only work within same filesystem
3. **Metadata Harmonization**: No automatic harmonization of different metadata schemas
4. **Performance**: Large cohort exports (100+ subjects, terabytes) may be slow with copy mode

### Known Issues

1. **Validation Coverage**: Not all BIDS specification aspects are validated
2. **Derivatives Support**: Limited support for derivative datasets
3. **Multi-Platform Export**: Exporting from Pennsieve + OpenNeuro may have inconsistencies
